01
Environment Setup
Install XAMPP, configure PHP, enable extensions

Install XAMPP (recommended for local dev)

  • Download from apachefriends.org — choose XAMPP 8.2+ for Windows/Mac/Linux
  • Install and launch the XAMPP Control Panel
  • Start Apache and MySQL modules
  • Place your project folder in C:\xampp\htdocs\autoverse\ (Windows) or /opt/lampp/htdocs/autoverse/ (Linux/Mac)

Verify PHP version and extensions

# Create test.php in your project folder:
<?php phpinfo(); ?>

# Visit: http://localhost/autoverse/test.php
# Confirm: PHP 8.1+, PDO enabled, PDO_MySQL enabled

Enable PHP extensions (if needed)

Open php.ini (in XAMPP: C:\xampp\php\php.ini) and uncomment:

extension=pdo_mysql
extension=openssl
extension=mbstring
✅ XAMPP already has PDO and PDO_MySQL enabled by default. You usually don't need to change anything.

Verify MySQL is running

Open your browser and go to http://localhost/phpmyadmin. You should see the phpMyAdmin dashboard.

02
Database Setup via phpMyAdmin
Create database, run schema, verify tables

Step-by-step in phpMyAdmin

  • Go to http://localhost/phpmyadmin
  • Click "New" in the left sidebar
  • Database name: autoverse_db · Collation: utf8mb4_unicode_ci · Click Create
  • Select autoverse_db from the left sidebar
  • Click the SQL tab at the top
  • Open your database/schema.sql file, copy all contents, paste into the SQL box
  • Click Go / Execute
✅ This creates all 5 tables and seeds 12 cars + 1 admin user automatically.

Verify tables were created

In phpMyAdmin left sidebar you should now see 5 tables:

TablePurposeKey Columns
usersRegistered user accountsid, name, email, password (bcrypt), role
carsAll car listingsid, brand, model, year, price, type, fuel, is_active
enquiriesContact / test drive requestsid, car_id, name, phone, status
favoritesUser wishlistuser_id, car_id (unique pair)

Update config/db.php credentials

define('DB_HOST',  'localhost');
define('DB_NAME',  'autoverse_db');
define('DB_USER',  'root');      // XAMPP default
define('DB_PASS',  '');          // XAMPP default = empty
define('APP_URL',  'http://localhost/autoverse'); // match your folder name
⚠️ Change DB_USER and DB_PASS before deploying to production. Never use root/empty in production.

Update Frontend (app.js)
Replace localStorage calls with PHP API fetch() calls

1. Set your API base URL

At the top of the new app.js, set this to your project URL:

const API_BASE = 'http://localhost/autoverse/api'; // ← change this

2. The apiFetch() helper

All API calls go through this wrapper that handles credentials, headers and error catching:

async function apiFetch(endpoint, options = {}) {
  const res = await fetch(`${API_BASE}/${endpoint}`, {
    credentials: 'include',   // ← CRITICAL: sends session cookie
    headers: {
      'Content-Type': 'application/json',
      ...(options.headers || {}),
    },
    ...options,
  });
  const data = await res.json();
  return { ok: res.ok, status: res.status, data };
}

4. Replace your old app.js

Copy the new frontend/app.js to your project and replace the old one. Your index.html and style.css remain completely unchanged.

✅ The new app.js is a drop-in replacement. Every function name stays the same — only the internals change from localStorage to fetch().

5. CORS is handled in config/db.php

// In setCorsHeaders() — called at top of every API file
$origin = $_SERVER['HTTP_ORIGIN'] ?? '';
if (in_array($origin, ALLOWED_ORIGINS)) {
    header("Access-Control-Allow-Origin: $origin");
}
header('Access-Control-Allow-Credentials: true'); // ← required for sessions

Add your frontend's origin to ALLOWED_ORIGINS in config/db.php if your HTML is served from a different port.

Change default admin password

In phpMyAdmin, run this SQL to update the password hash:

-- In phpMyAdmin → autoverse_db → SQL tab:
UPDATE users
SET password = '$2y$12$HASH_HERE'
WHERE email = 'admin@autoverse.in';

-- Generate a bcrypt hash with PHP:
-- <?php echo password_hash('YourNewPassword123', PASSWORD_BCRYPT); ?>

Create additional admin users

INSERT INTO users (name, email, password, role, is_verified)
VALUES (
  'Manager Name',
  'manager@autoverse.in',
  '$2y$12$...bcrypt_hash...',  -- use password_hash() to generate
  'admin',
  1
);
⚠️ The admin panel checks $_SESSION['user']['role'] === 'admin' on every page load. Non-admin users are redirected to login automatically.

Deploy to Production
Shared hosting (cPanel), VPS, or cloud

Option A — Shared Hosting (Hostinger / GoDaddy / Bluehost)

  • Upload all files via cPanel File Manager or FTP (FileZilla) to public_html/
  • In cPanel → MySQL Databases → Create database + user + assign all privileges
  • In cPanel → phpMyAdmin → Select your new DB → SQL → paste schema.sql
  • Update config/db.php: DB_HOST, DB_NAME, DB_USER, DB_PASS to match cPanel values
  • Update APP_URL to your actual domain: https://yourdomain.com
  • Update API_BASE in frontend/app.js to match

Option B — VPS (DigitalOcean / AWS Lightsail)

# Install on Ubuntu 22.04
sudo apt update
sudo apt install apache2 php8.2 php8.2-mysql mysql-server php8.2-mbstring -y

# Enable mod_rewrite
sudo a2enmod rewrite

# Clone/upload your project
sudo cp -r autoverse/ /var/www/html/

# Set permissions
sudo chown -R www-data:www-data /var/www/html/autoverse

# Create database
sudo mysql -e "CREATE DATABASE autoverse_db;"
sudo mysql autoverse_db < /var/www/html/autoverse/database/schema.sql
